3//! # Overview
4//! Envelopes are functions over time used to modulate signal parameters such
5//! as amplitude or frequency. A common envelope is ADSR (Attack-Decay-Sustain-Release),
6//! defined by four stages where amplitude evolves over time. ADSR is typically
7//! implemented using linear or exponential segments.
8//!
9//! # Mathematical note
10//! For an ADSR envelope the amplitude can be represented piecewise. For example,
11//! - Attack (0 -> A) over t_a seconds
12//! - Decay (A -> S) over t_d seconds
13//! - Sustain S for the note duration
14//! - Release (S -> 0) over t_r seconds after note off
15//!
16//! Implementations should map `time` and `note_off` to the appropriate stage
17//! and return a normalized amplitude in `[0.0, 1.0]`.

24/// An envelope that can be used to modulate a signal over time.
25/// The base principle is simply to have a function with a varying value over time.
26/// This value can then be used to shape either the amplitude, frequency or any other parameter of a sound.
27#[typetag::serde]
28pub trait Envelope: std::fmt::Display + std::fmt::Debug + Send + Sync {
29    /// Returns the envelope value at the given point in time. The timestamps
30    /// is expected to be mapped to the envelope's duration, that is the
31    /// minimum value is 0.0.
32    fn at(&self, time: f32, note_off: f32) -> f32;
33
34    /// Returns whether the envelope has completed or not based on the
35    /// current time & note_off timestamp
36    fn completed(&self, time: f32, note_off: f32) -> bool;
37}
