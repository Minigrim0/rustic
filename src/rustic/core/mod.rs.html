1//! Core audio processing module
2//!
3//! ## Overview
4//! The `core` module provides the foundational DSP abstractions and primitives
5//! used across the workspace: `Generator`s (oscillators and tone combiners),
6//! `Envelope`s (ADSR and segment-based curves), `Filter`s (frequency and time
7//! domain processors), and graph utilities for routing signals between nodes.
8//!
9//! ## Purpose
10//! This module exists to expose composable, well-tested building blocks so
11//! frontends and higher-level APIs (in `instruments` and `score`) can assemble
12//! musical behaviours without re-implementing low-level DSP logic.
13//!
14//! ## Key components
15//! - `generator`: Oscillators, noise sources and multi-tone combiners.
16//! - `envelope`: Time-based modulation (ADSR, segments) that shapes amplitude or
17//!   other parameters.
18//! - `filters`: Audio processors (amplifiers, tremolo, delays, resonant filters).
19//! - `graph`: Utilities to wire processors together and build signal graphs.
20//! - `utils`: Shared types, e.g. `Note` and tone frequency tables.
21//!
22//! ## Usage
23//! Prefer using `core::prelude` for common types. See module docs for examples
24//! and mathematical foundations (e.g., sine wave generation and ADSR equations).
25
26/// Defines the different envelope shapes & types
27/// Envelopes implement the `Envelope` trait and can
28/// be of 3 types; linear, bezier, adsr
29pub mod envelope;
30
31/// Filters are structures that implement the `Filter` trait. They
32/// operate on audio signals to modify their frequency response.
33/// examples include low-pass, high-pass, tremolo, ...
34pub mod filters;
35
36/// Generators are structures that implement the `Generator` trait.
37/// They generate audio signals of different types such as sine, square, sawtooth, etc.
38pub mod generator;
39
40/// Audio signal graph processing and routing
41pub mod graph;
42
